CAN Bus Data & Vehicle Forensics

Understanding CAN bus data is essential for vehicle forensics, diagnostics, and security investigations.

What is CAN Bus?

The Controller Area Network (CAN) is a communication protocol used in modern vehicles for real-time data exchange between different electronic control units (ECUs). The CAN Bus network shares similarities with OBD data. Both types of data are accessed through the OBD port, described in the previous section, however CAN data provides a much more in depth insight into the state of a vehicle, and provides investigators with data such as individual wheel speeds, brake pressure, steering angle, and a range of other data that OBD won't show.

Although CAN data provides us with a more in depth picture, this also comes with it's own challenges. Firstly, the CAN network requires specialised tools to access data such as CANEdge or SavvyCAN. Another challenge is that CAN data is not standardised and therefore not human readable, like OBD data is, which means that investigators must use Database CAN (DBC) files in order to decode the raw data stream. Another challenge with CAN data is that the CAN bus network transmits thousands of messages per second, the majority of which are not relevant to a forensic investigation, therefore these messages must be filtered in order to extract the relevant information.
